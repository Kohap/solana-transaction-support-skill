# Support Privacy Rules

Apply these rules whenever transaction support, wallet reports, or user screenshots are involved.

- Never ask for seed phrases, private keys, recovery phrases, raw auth tokens, or wallet login details.
- Never suggest that a user signs an arbitrary transaction for debugging.
- Sanitize public reports before posting them.
- Separate public customer replies from private engineering notes.
- Escalate immediately if evidence suggests unauthorized fund movement, compromised authorities, malicious frontend behavior, or private key exposure.
